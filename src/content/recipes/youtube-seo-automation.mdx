# Why this recipe exists

Every month, I have ~30 YouTube videos that need title and description tuning based on actual CTR data. Doing it manually means:

- Open YouTube Studio
- Filter to videos with declining CTR
- For each one: read recent comments, check competing video titles, draft a better title, save it, repeat
- About 4 minutes per video × 30 videos = 2 hours

That's 24 hours a year on a task that's mostly mechanical. So I chained three MCPs and got it down to 8 minutes.

## The architecture

```
                 ┌──────────────┐
                 │   Claude     │  ← orchestrator
                 │  (the brain) │
                 └──────┬───────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
┌───────▼─────┐  ┌──────▼──────┐  ┌────▼──────────┐
│ youtube-mcp │  │ playwright- │  │  cloudflare-  │
│             │  │     mcp     │  │      mcp      │
└─────────────┘  └─────────────┘  └───────────────┘
```

Three MCPs:

- **youtube-mcp** — fetches video metadata, CTR data, posts updates
- **playwright-mcp** — for the steps YouTube API doesn't expose (some metadata is web-only)
- **cloudflare-mcp** — purges blog cache where embedded videos are referenced

## The flow

1. **Pull video list.** youtube-mcp → "list my last 30 videos with CTR data"
2. **Filter to candidates.** Claude looks at the data, picks the videos with declining CTR over the past 30 days.
3. **Generate new titles.** For each candidate, Claude generates 3 alternative titles based on:
   - The current title (what we're improving)
   - The video transcript snippet (what's the video actually about)
   - Top-performing similar videos (what works in this niche)
4. **Pick the winner.** Claude scores each alternative against my brand voice rules and picks one.
5. **Push to YouTube.** youtube-mcp → "update title for video X to Y"
6. **Update blog references.** If the video is embedded in a blog post, also update the post title.
7. **Purge Cloudflare cache.** cloudflare-mcp → "purge cache for /blog/* and /videos/*".
8. **Generate diff report.** Markdown report of all changes for human review.

## Step-by-step build

### Prerequisites

- Claude Pro account (or any model that supports MCP tool calling)
- youtube-mcp installed and OAuth-connected to your channel
- playwright-mcp installed
- cloudflare-mcp installed with API token in scope `Cache:Purge` only
- A Markdown file with your brand voice rules (mine is `~/voice-rules.md`)

### The orchestration prompt

Save this as `~/recipes/youtube-seo.md`:

```markdown
You are a YouTube SEO specialist for my channel.

For each of my last 30 videos:
1. Use youtube-mcp:list_videos to fetch them with CTR + impression data.
2. Filter to videos where CTR has declined >15% over the past 30 days.
3. For each, generate 3 alternative titles using the transcript snippet and these voice rules:
   [paste content of ~/voice-rules.md]
4. Score each alternative and pick the winner.
5. Show me a diff of (current title) → (new title) for all candidates.
6. STOP and wait for my approval before pushing changes.
7. After my approval: use youtube-mcp:update_video for each.
8. Use cloudflare-mcp:purge_cache for /blog/* and /videos/*.
9. Output a markdown report summarising changes.
```

### Run it

```bash
# Save the prompt
echo "..." > ~/recipes/youtube-seo.md

# Run via Claude Desktop or a script
claude < ~/recipes/youtube-seo.md
```

Estimated runtime: ~8 minutes.

## The catch

Three things that bit me on early runs:

1. **YouTube rate limits.** 50 video updates / day max. If your channel is bigger, batch across days.
2. **Title language drift.** Claude sometimes reduces character count to fit YouTube's 100-char limit, but in doing so drops emojis I had used deliberately. **Fix:** add to the prompt: *"Preserve emojis from the original title unless removing them objectively improves CTR."*
3. **Cache purge timing.** Cloudflare's cache purge takes ~30 seconds on the free tier. Don't refresh blog posts immediately after.

## My take

I run this monthly. The first run caught two typos I'd missed manually for three months. The review step is non-negotiable — Claude gets ~95% right, but the 5% includes edge cases (emoji handling, brand-specific phrasing) that I want to eyeball before publishing.

Would I run this again? **Yes — every month.** It pays for itself in 5 minutes.

## Variants

- **Replace youtube-mcp with x-mcp** for Twitter/X title-equivalent (post text) optimisation
- **Drop cloudflare-mcp** if you don't have a blog with embedded videos
- **Add slack-mcp** to post the diff report to a channel for team review
- **Add validate-mcp** before push to enforce "title contains keyword X"
